2.2 定时 Cron（GitHub 使用 UTC）

报告Cron (UTC)收件约 (JST)--mode调 Gemini
日报0 21 * * 1-5周二～周六 06:00daily否*
周报0 3 * * 6周六 12:00weekly是
月报0 21 28-31 * *月末最后工作日附近monthly是
季报0 5 31 1,7,10 * + 0 5 30 4 *1/4/7/10 月末 ~14:00quarterly是
年报0 5 31 12 *12/31 ~14:00annual是
盘中狙击0,30 14-19 * * 2-623:00～次日 04:30 每 30 分—命中时

*日报默认不调 Gemini；命中狙击条件时例外调用 Flash 发加急信。月报 cron 触发后脚本内 is_last_trading_day_of_month() 校验，非月末工作日则跳过（可用 --force）。日报在 cron 触发后还会校验当日美东 session 是否为 NYSE 实际成交日（SPY 日 K）；法定节假日/休市则跳过，避免发出「尸检式」空报。

2.3 为什么日报是 JST「周二～周六」而非「周一～周五」？

日报对齐的是美股每个交易日收盘后，不是日本工作日历。

UTC 触发JST 收信覆盖的美股交易日
周一 21:00周二 06:00美股周一收盘后
周二 21:00周三 06:00美股周二收盘后
周三 21:00周四 06:00美股周三收盘后
周四 21:00周五 06:00美股周四收盘后
周五 21:00周六 06:00美股周五收盘后

若强行改为 JST 周一至周五发报，会漏掉周五美股收盘报，并在周一早上发出无新交易日的空报。

2.4 美股休市门禁（日报专用）

Cron 按 UTC 周一～周五触发，但不等于美股每个自然日都开盘。脚本 should_run_daily_report()（src/lynch/market_calendar.py）在 --mode daily 时执行：

  • 以 cron 触发时刻映射到美东日历日，作为应对齐的 session；
  • 用 SPY 日 K 索引判断该日是否实际成交（覆盖 NYSE 法定节假日、暂停交易；早收日仍视为成交日）；
  • 休市则打印 日报跳过：美股休市（美东 YYYY-MM-DD …） 并不发邮件；
  • 正常交易日简报页眉标注「数据对齐：美东 YYYY-MM-DD 收盘」。

调试可 --force 绕过门禁。SPY 日历拉取失败时保守放行（仅排除周末），避免 Actions 偶发网络问题导致长期停报。

2.5 定时任务运维要求

手动 Run workflow ≠ 自动定时。Actions 历史若全部为 Manually run by …，说明 cron 从未触发，不会自动收信。

启用步骤：Actions → 对应 Workflow → 若顶部有 Enable scheduled workflows 须点击启用； Settings → Actions → 允许 Actions 运行。定时任务仅对 main 分支上的 workflow 文件生效。

GitHub 定时触发后默认参数：scope=full，market=US（可用仓库 Variable MARKET 覆盖）。

6. 量化引擎 (metrics.py)

6.1 股息修正 PEG

PEG = P/E ÷ (长期CAGR% + 股息率%)
P/E 取值：优先 valuation_pe（季报/年报锚定），否则 trailing_pe
CAGR：多年 EPS 或净利润 CAGR；拒绝 yfinance 单季 earningsGrowth
增速 > 50% → 分母封顶 35%

6.2 五项排雷

key规则豁免
peg≤0.5 极佳；≤1 合理；>2 高估周期股高 PEG → 黄灯
debt长期负债/权益 ≤33%金融股
inventory存货 YoY − 销售 YoY科技 / 存货<5% 总资产
net_cash每股净现金/股价—
fcfFCF 正负 + FCF/市值—

6.3 SBI / NISA 可交易

  • 美股：纽交所/纳斯达克主板 + 市值 ≥ 3 亿美元；排除 OTC、5 位 F/Y ADR。
  • 日股 .T：技术上可交易，但默认流水线已不扫描（MARKET=US）。

7. 双层漏斗 (funnel.py)

  • 第一层：并发 get_quick_screen；PEG≤1.5 或 净现金比≥0.30，且负债≤0.50。
  • 第二层：watchlist 永远 AI；其余按 PEG/净现金排序，前 MAX_AI_ANALYSIS_COUNT（默认 30）送 Gemini。
  • fatal_warnings → 邮件置顶红灯；is_quality_pick → 优质股（≤20）；cyclical_watch → 周期观察。
  • check_daily_sniper_trigger：daily + full 幸存池 + SBI + 日 K 跌>5% + PEG<0.5。

8. AI 编排

  1. get_fundamentals(ticker, mode=report_mode)
  2. compute_metrics(f)
  3. build_data_block — 颗粒度区块置顶，含 spot 与锚定价双轨展示
  4. 注入 get_mode_context(mode)、story_diff_context（有历史时）、RAG
  5. llm.generate → 四步分析 + 文末唯一【行动指令】
  6. extract_signal — 取最后一条指令行；Python 不 downgrade
  7. append_record → data/history/YYYY-MM-DD.jsonl

模型默认 gemini-2.5-flash；狙击演练 SNIPER_DRILL_MAX_TOKENS=2048。

14. 非功能性需求

  • 容错：单股失败不中断；错误渲染为「分析失败：{原因}」块（如 temporal 字段冲突已修复）。
  • 成本：daily 零 Gemini（狙击命中除外）；AI 硬顶 30 只/次。
  • 限流：Yahoo 重试 + SEC 抽样 + 并发 8；Actions timeout 30min / 狙击 15min。
  • 时区：简报时间戳 Asia/Tokyo；Cron 配置 UTC。
  • 信任：裁决看板直接反映 Gemini 【行动指令】；无 Python valuation veto。
