彼得·林奇 Lynch Agent — 系统需求文档

文档版本：与代码库 main 分支同步（2026-07）
项目路径：stock-monitor / GitHub nanxv/Lynch
定位：双层漏斗 + 立体化投研周期 + 盘中狙击 + 多维时间轴估值对齐
铁律：数据为王，不预测宏观；AI 裁决不被 Python 二次篡改

1. 系统总览

本系统是一套彼得·林奇式自动化投研流水线：从全市场/自选股清单出发，经 Yahoo Finance 抓取财务与行情，本地计算林奇硬指标，按报告周期注入不同颗粒度数据，选择性调用 Google Gemini 生成四步会诊，最终组装 Markdown 简报经 Gmail SMTP 发送。

3. 数据供应层需求

3.1 数据源工厂

  • 环境变量 DATA_PROVIDER，默认 yahoo（yfinance）。
  • 预留 fmp、jquants 接口，未实现时抛 NotImplementedError。
  • 单例缓存：get_fundamentals(ticker, mode=...) 键为 {TICKER}::{mode}。

3.2 Yahoo Finance 抓取

方法API用途
get_quick_screenticker.info 单次第一层漏斗：粗 PEG、负债率、净现金比、SBI 可交易
_fetch_fundamentalsinfo + 三张年表完整林奇 SOP 数据区块
get_daily_price_changehistory(5d, 1d)最近交易日 vs 前一交易日（日报狙击）
get_stock_price_changehistory(period, 1d)区间涨跌幅（日报展示 5 日）
get_intraday_snapshotregularMarketPrice, previousClose盘中狙击：即时跌幅、即时 PEG、5 年最低 P/E

反爬：优先 curl_cffi 浏览器指纹；.info 失败退避重试 3 次。

3.3 成分股宇宙 (universe)

  • us：SEC company_tickers.json，纯字母代码；超样本量时随机无放回抽样（默认 500）。
  • sp500 / nasdaq100：维基百科 HTML 表。
  • jpx：东证 Excel 全量（约 4000 只）。
  • 去重 + correct_ticker 纠错后截断至 MAX_UNIVERSE_SCAN（默认 1200）。

3.4 多维时间轴 (temporal) — 防时空错乱

report_mode额外数据估值规则
daily / weeklyspot 价 + 可选 5 年 P/E 区间TTM P/E + 即时价算 PEG
monthly约 1 个月前收盘价 → 反推 P/E、PEG；RSI、20 日涨跌对比估值扩张/收缩
quarterlyquarterly_income_stmt + quarterly_balance_sheet；QoQ / 单季 YoY财报截止日后 3 交易日均收为锚定价；PEG 用锚定 P/E
annual5–10 年营收/利润/回购/股息；5 年最低/平均 P/E年终清仓 + 历史低估水位

Fundamentals 扩展字段：spot_price, valuation_anchor_*, price_1mo_ago, pe_5y_min/avg 等，经 granularity_block 置顶注入 LLM。

4. 量化参数需求 (metrics.py)

4.1 股息修正 PEG

PEG = trailing_P/E ÷ (长期CAGR% + 股息率%)
长期 CAGR：优先多年摊薄 EPS CAGR，其次净利润 CAGR；拒绝单季度 earningsGrowth
增速 > 50% 时分母封顶 35%
季报/年报硬指标：若存在 valuation_pe，则用锚定 P/E 替代 trailing_pe 算 PEG

4.2 排雷指标 (Metric)

key规则豁免
peg≤0.5 极佳；≤1 合理；>2 高估周期股高 PEG 黄色警告
debt长期负债/权益 ≤33% 绿金融股完全豁免
inventory存货 YoY − 销售 YoY科技/轻资产(<5% 总资产)豁免
net_cash每股净现金 / 股价—
fcf自由现金流正负 + FCF/市值—

4.3 SBI / NISA 可交易

  • 日股 .T：可交易。
  • 美股：纽交所/纳斯达克主板 + 市值 ≥ 3 亿美元；排除 OTC、5 位 F/Y 结尾 ADR。

4.4 公司类型初判 (classify_company)

隐蔽资产型 / 困境反转型 / 周期型 / 快速增长型 / 缓慢增长型 / 稳定增长型 — 供 LLM Step 1 复核。

5. 双层漏斗需求 (funnel.py)

5.1 第一层（纯代码，无 Gemini）

  • 并发 SCAN_WORKERS（默认 8）调用 get_quick_screen。
  • 通过条件（其一且负债不超标）：quick_peg ≤ 1.5 或 net_cash_ratio ≥ 0.30。
  • 负债刷掉：debt_ratio > 0.50。

5.2 第二层（AI 成本熔断）

  • watchlist.yaml 标的标记 is_priority=True，永远进 AI 组且优先。
  • 其余幸存者按 AI_SORT_KEY（peg 升序 / net_cash 降序）排序，取前 MAX_AI_ANALYSIS_COUNT（默认 30）。
  • 超额部分仅硬指标，不调 Gemini。

5.3 衍生筛选块

  • fatal_warnings：存货暴增、负债超标、盈利暴跌（周期股部分豁免）→ 邮件置顶红灯。
  • is_quality_pick：无红灯 + PEG≤1 + 低负债 + 正 FCF → 优质股推荐（最多 20 只）。
  • cyclical_watch：周期股底部观察理由。
  • check_daily_sniper_trigger：SBI + 单日跌 >5% + PEG<0.5（日报全市场 SBI 池）。

6. AI 编排需求 (agent.py + llm.py)

6.1 analyze_company 流程

  1. provider.get_fundamentals(ticker, mode=report_mode)
  2. compute_metrics(f) → 硬指标
  3. build_data_block(f, m)：颗粒度区块置顶 + 财务序列 + 排雷结果
  4. 若 data_only=False：注入 mode_context、story_diff_context、RAG 片段
  5. llm.generate(system_prompt, user_content) → 四步分析 + 【行动指令】
  6. 返回 LynchAnalysis（含 narrative）

6.2 Gemini 配置

  • GEMINI_API_KEY 必填（AI 模式）；模型默认 gemini-2.5-flash，可 GEMINI_MODEL 覆盖。
  • MODE_CONTEXT：weekly / monthly / quarterly / annual 各有专项 Prompt（见 llm.py）。
  • 狙击演练：SNIPER_DRILL_MAX_TOKENS=2048。

6.3 信号解析 (signals.py)

  • 取 narrative 最后一条【行动指令】行。
  • 优先级：🟢 强买(0) → 🟡 观察(1) → ⚪ 持有(2) → 🔴 卖出(3)。
  • 禁止 Python 二次篡改 AI 裁决（无 valuation veto）。

6.4 故事线存档 (history.py)

  • 每次分析后追加 data/history/YYYY-MM-DD.jsonl：ticker、裁决、PEG、负债、存货差。
  • AI 模式分析前 load_previous(ticker) → build_story_diff_context 强制 Step 1 前一句话：好转/持平/恶化。

8. 狙击雷达需求

8.1 收盘后日报狙击

  • 触发：daily + scope=full 幸存池中 sbi_tradable + 日 K 跌幅 >5% + PEG<0.5。
  • 流程：run_sniper_alert → Gemini 两分钟演练 → 仅 🟢 强买 → send_sniper_alert。
  • 标题：【🚨林奇狙击警报】{代码} 现价已杀入绝佳特价期！

8.2 盘中实时狙击

  • Workflow：.github/workflows/sniper_realtime.yml，UTC 周二~周六 14:00–19:30 每 30 分钟。
  • 脚本：run_realtime_sniper.py，仅 watchlist.yaml 且 SBI 可交易。
  • 跌幅：(regularMarketPrice / previousClose) - 1 ≤ -5%；PEG：即时估算 < 0.5。
  • 附：5 年历史最低隐含 P/E。
  • 防刷：data/sniper_cache/YYYY-MM-DD.json，同股同日最多 1 封。
  • 标题：【🚨林奇深夜特快】{代码} - {简称} 盘中突发暴跌 [{跌幅}]！速去 SBI 账户护航！

9. 简报组装与发送 (notify.py + run_scheduled_analysis)

9.1 邮件结构（自上而下）

  1. 页眉：模式标题、JST 时间、分析只数（AI / 硬指标）、漏斗统计。
  2. 置顶摘要块（blockquote 卡片样式）：
    • 🟢 推荐深挖优质股（PEG 升序，≤20）
    • 🔴 致命红灯排雷
    • 🧠 智能体最终裁决看板（双轨：SBI 免税直通车 / 硬核场外；强买按 PEG+FCF 排序）
    • 🌀 周期股低谷观察
  3. 详情区双轨：
    • 第一章：🏦 主板免税成分股会诊详情（sbi_tradable=True）
    • 第二章：🌀 硬核场外深挖（sbi_tradable=False，AI 模式）

9.2 邮件主题动态标签

可选追加：🏦N只SBI强买 · 🧠N只强买 · 🟢N只优质 · 🔴N只排雷 · 🌀N只周期

9.3 SMTP 需求

变量说明
SMTP_SERVER / SMTP_PORT如 smtp.gmail.com : 587
SMTP_USERNAME / SMTP_PASSWORD应用专用密码

双格式：纯文本 + HTML（Markdown 转 HTML，blockquote 卡片 CSS）。未配置时优雅降级，不崩溃。

13. 非功能性需求

  • 容错：单股失败不中断流水线；FundamentalsError / LLMError 捕获后渲染错误块。
  • 成本：daily 默认零 Gemini；AI 硬上限；狙击仅命中才调 Flash。
  • 限流：Yahoo 重试 + 抽样 + 并发上限；GitHub Actions timeout 30min（狙击 15min）。
  • 时区：简报时间戳统一 Asia/Tokyo (JST)。
  • 信任模型：AI 【行动指令】为最终裁决来源；Python 不做 downgrade veto。

14. 必看列表 (watchlist.yaml)

结构：stocks[].ticker / name / market / tier / note。在 scope=watchlist 或全市场漏斗中均标记 is_priority，保证必看 + 优先 AI。
