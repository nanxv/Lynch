# 铁律 2.5 股票监控 — 使用说明

## 这个工具是干什么的？

它会自动帮你盯股票，按「铁律 2.5」规则判断：

- **高位排雷**：股价离 10 周均线太远（情绪过热）→ 不提醒
- **核心池 (Tier 1)**：贴近 10 周均线 → 发「核心阻击」警报
- **常规池 (Tier 2)**：大盘跌势 + 跌破 20 日均线 → 发「常规买入」警报

有信号时，**Telegram 推送到手机**。同一股票 24 小时内不会重复轰炸。

---

## 你每天需要做什么？

**几乎不用动手。** 装好定时任务后：

| 时间 (东京) | 做什么 |
|-------------|--------|
| 08:30 | 扫描日股，宏观筛选 |
| 14:30 | 扫描日股，战术触发 |
| 04:30 / 05:30 | 扫描美股（收盘前 30 分钟，睡觉时自动跑） |

你只需：**看手机 Telegram → 有警报再决定是否去 SBI 下单**。

警报底部会提示：核对现金流，资金不够可考虑 S 株 / 碎股。

---

## 收到警报后怎么读？

示例：

```
🚨 核心阻击 | 信越化学 (4063.T)
类别: Tier 1 | 市场: JP
现价: ¥7,307
10周均线: ¥7,200 (偏离 +1.5%)
20日均线: ¥7,334 (偏离 -0.4%)
```

- **核心阻击** = 核心池，周线回踩，可重点看
- **常规买入** = 常规池，大盘弱 + 日线破位，偏战术
- **偏离** = 现价相对均线的百分比

## 怎么改监控的股票？

编辑 `watchlist.yaml`：

```yaml
stocks:
  - ticker: "4063.T"    # 代码：日股加 .T，美股直接写符号
    name: "信越化学"
    market: JP          # JP 或 US
    tier: 1             # 1=核心池，2=常规池
    note: "备注"
```

改完保存即可，下次扫描自动生效。

## 两个人一起收 Telegram（夫妻共用）

**方式 A — 推荐：各收各的（最简单）**

1. 老婆在 Telegram 里找到你们的机器人，发一条 `/start` 或任意消息
2. 浏览器打开（把 `TOKEN` 换成真实 token）：
   ```
   https://api.telegram.org/bot<TOKEN>/getUpdates
   ```
3. 找到她的 `"chat":{"id": 数字 }`，记为她的 chat_id
4. 在你的 `.env` 里加上（逗号分隔）：
   ```
   TELEGRAM_CHAT_IDS=你的id,老婆的id
   ```
5. 保存后跑一次测试：
   ```bash
   .venv/bin/python -m src.main --market JP
   ```
   两个人手机都应收到。

**方式 B — 建一个 Telegram 群**

1. 建群「家庭股票警报」，把机器人和你们都拉进去
2. 用群的 chat_id（通常是负数）填进 `.env`
3. 信号发到群里，两人都能看到

## 常见问题

**Q：为什么同一只股票没再推 Telegram？**  
A：24 小时去重。终端里仍能看到信号，标了 `🔕 Telegram 跳过`。

**Q：半夜美股扫描 Mac 要开着吗？**  
A：要。插电 + 勿睡眠，或用「节能」里允许定时唤醒。

**Q：数据从哪来？**  
A：Yahoo Finance（`yfinance`），免费，偶有延迟。

**Q：会自动帮我在 SBI 下单吗？**  
A：**不会。** 只提醒，买卖你自己在 SBI 操作。

## 彼得·林奇分析 Agent（深度基本面分析）

监控只告诉你「什么时候价格到位」，**林奇 Agent 告诉你「这家公司值不值得买」**。
它严格按《彼得·林奇的成功投资》的 SOP 做：定性分类（六大类）+ 定量排雷（PEG、
负债率、存货 vs 销售、每股净现金、自由现金流），数据全部来自 Yahoo Finance（美股+日股皆可），
再由 Claude 用林奇的口吻给出四步分析与裁决。

### 配置（一次性）

```bash
# 装依赖（新增 anthropic）
.venv/bin/pip install -r requirements.txt
# 在 .env 里填 Claude key（到 https://console.anthropic.com/ 生成）
ANTHROPIC_API_KEY=sk-ant-...
```

### 用法

```bash
# 完整分析（调用 Claude 生成林奇式叙述）
.venv/bin/python scripts/analyze.py AMD

# 带上你的买入动机（会触发「特定产品占比」排查）
.venv/bin/python scripts/analyze.py 4063.T --note "因为我天天用它家的半导体材料"

# 只看硬指标，不花 Claude 的钱（无 key 时自动进此模式）
.venv/bin/python scripts/analyze.py 6859.T --data-only
```

日股加 `.T`（如 `4063.T`），美股直接写符号（如 `AMD`、`KO`）。

### 输出读法

- **🟢/🟡/🔴** 三色灯是林奇量化排雷的判定：绿=通过，黄=警惕/数据不足，红=危险。
- **PEG**：市盈率 ÷ 增长率。≤1 合理，≤0.5 极佳，>2 危险。
- **长期负债/股东权益**：超过 1/3（0.33）就要留意。
- **存货 vs 销售**：存货增速高于销售增速→红灯（货卖不动）。
- **每股净现金**：（现金-总负债）/ 总股本，越厚安全垫越足。

> 铁律：拿不到真实市盈率/负债/存货数据前，Agent 绝不下「值得投资」的结论；
> 也从不预测宏观经济，只回到公司基本面。

### 全自动后台流水线（GitHub Actions + 邮件，无需开电脑）

系统自动发两类**邮件**（Markdown 排版）：

| 模式 | 触发时间(JST) | 内容 | 邮件标题 |
|------|--------------|------|----------|
| **日报 daily** | 周二~周六 06:00 | 仅量化过滤，监控每日股价/估值异动 | `【彼得林奇自选股监控】日报 - 年月日` |
| **周报 weekly** | 周六 12:00 | 全套 Claude 四步叙述+裁决，置顶 🔴 红灯排雷 | `【彼得林奇深度分析】周报 - 年月日` |

**配置步骤（一次性）：**

1. 把项目推到 GitHub 仓库。
2. 仓库 **Settings → Secrets and variables → Actions → New repository secret**，添加：
   - `ANTHROPIC_API_KEY`（周报需要）
   - `SMTP_SERVER`、`SMTP_PORT`、`SMTP_USERNAME`、`SMTP_PASSWORD`、`RECEIVER_EMAIL`（收发邮件必填）
   - `ANTHROPIC_MODEL`（选填，默认 `claude-sonnet-4-5`）
3. 完成。`.github/workflows/run_analysis.yml` 会自动按上表时间运行，也可在 Actions 页
   点 **Run workflow** 手动触发（可选 daily/weekly）。

> 密钥只从系统环境变量读取，**不需要**把 `.env` 提交到仓库（`.env` 已在 `.gitignore` 里）。

**本地手动跑同一条流水线：**

```bash
# 日报（仅量化，不发邮件只打印）
.venv/bin/python scripts/run_scheduled_analysis.py --mode daily --no-email

# 周报（全套 Claude），发邮件（需已配 SMTP）
.venv/bin/python scripts/run_scheduled_analysis.py --mode weekly

# 临时指定几只
.venv/bin/python scripts/run_scheduled_analysis.py --mode daily --tickers AMD KO --no-email
```

调度时间在 `.github/workflows/run_analysis.yml` 里改（cron 用 UTC）：
`0 21 * * 1-5` 为日报，`0 3 * * 6` 为周报。

#### 怎么拿邮箱 SMTP 授权码？

- **Gmail**：`SMTP_SERVER=smtp.gmail.com`、`SMTP_PORT=587`。先在 Google 账号开启
  **两步验证**，再到「安全性 → 应用专用密码」生成 16 位密码，填入 `SMTP_PASSWORD`（不是登录密码）。
- **QQ 邮箱**：`smtp.qq.com`、端口 `465`。设置 → 账户 → 开启 SMTP → 生成**授权码**。
- **Outlook/163** 同理：开启 SMTP 服务后使用授权码。`SMTP_USERNAME` 填完整发件邮箱地址。
