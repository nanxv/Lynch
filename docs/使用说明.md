# 铁律 2.5 股票监控 — 使用说明

> 本仓库含两套子系统：**铁律 2.5 均线监控**（本文前半）与 **彼得·林奇 Lynch Agent**（本文后半）。
> 总览见 [README.md](../README.md)；Lynch 技术细节见 [lynch-requirements.html](lynch-requirements.html)。

## 这个工具是干什么的？

它会自动帮你盯股票，按「铁律 2.5」规则判断：

- **高位排雷**：股价离 10 周均线太远（情绪过热）→ 不提醒
- **核心池 (Tier 1)**：贴近 10 周均线 → 发「核心阻击」警报
- **常规池 (Tier 2)**：大盘跌势 + 跌破 20 日均线 → 发「常规买入」警报

有信号时，**Telegram 推送到手机**。同一股票 24 小时内不会重复轰炸。

## 你每天需要做什么？

**几乎不用动手。** 装好定时任务后：

| 时间 (东京) | 做什么 |
|-------------|--------|
| 08:30 | 扫描日股，宏观筛选 |
| 14:30 | 扫描日股，战术触发 |
| 04:30 / 05:30 | 扫描美股（收盘前 30 分钟，睡觉时自动跑） |

你只需：**看手机 Telegram → 有警报再决定是否去 SBI 下单**。

警报底部会提示：核对现金流，资金不够可考虑 S 株 / 碎股。

---

## 收到警报后怎么读？

示例：

```
🚨 核心阻击 | 信越化学 (4063.T)
类别: Tier 1 | 市场: JP
现价: ¥7,307
10周均线: ¥7,200 (偏离 +1.5%)
20日均线: ¥7,334 (偏离 -0.4%)
```

- **核心阻击** = 核心池，周线回踩，可重点看
- **常规买入** = 常规池，大盘弱 + 日线破位，偏战术
- **偏离** = 现价相对均线的百分比

编辑 `watchlist.yaml`：

```yaml
stocks:
  - ticker: "4063.T"    # 代码：日股加 .T，美股直接写符号
```

改完保存即可，下次扫描自动生效。

## 两个人一起收 Telegram（夫妻共用）

**方式 A — 推荐：各收各的（最简单）**

1. 老婆在 Telegram 里找到你们的机器人，发一条 `/start` 或任意消息
2. 浏览器打开（把 `TOKEN` 换成真实 token）：
   ```
   https://api.telegram.org/bot<TOKEN>/getUpdates
   ```
3. 找到她的 `"chat":{"id": 数字 }`，记为她的 chat_id
4. 在你的 `.env` 里加上（逗号分隔）：
   ```
   TELEGRAM_CHAT_IDS=你的id,老婆的id
   ```
5. 保存后跑一次测试：
   ```bash
   .venv/bin/python -m src.main --market JP
   ```
   两个人手机都应收到。

**方式 B — 建一个 Telegram 群**

1. 建群「家庭股票警报」，把机器人和你们都拉进去
2. 用群的 chat_id（通常是负数）填进 `.env`
3. 信号发到群里，两人都能看到

## 常见问题

**Q：为什么同一只股票没再推 Telegram？**  
A：24 小时去重。终端里仍能看到信号，标了 `🔕 Telegram 跳过`。

**Q：半夜美股扫描 Mac 要开着吗？**  
A：要。插电 + 勿睡眠，或用「节能」里允许定时唤醒。

**Q：数据从哪来？**  
A：铁律 2.5 监控用 Yahoo Finance（`yfinance`）。林奇 Agent 生产环境默认 **FMP stable API**（`DATA_PROVIDER=fmp`），受限时回退 Yahoo。

**Q：会自动帮我在 SBI 下单吗？**  
A：**不会。** 只提醒，买卖你自己在 SBI 操作。

## 彼得·林奇分析 Agent（深度基本面分析）

监控只告诉你「什么时候价格到位」，**林奇 Agent 告诉你「这家公司值不值得买」**。
它严格按《彼得·林奇的成功投资》的 SOP 做：定性分类（六大类）+ 定量排雷（PEG、
负债率、存货 vs 销售、每股净现金、自由现金流），再由 **Google Gemini** 用林奇的口吻给出四步分析与裁决。

### 数据源

| 提供商 | 环境变量 | 说明 |
|--------|----------|------|
| **FMP**（生产推荐） | `DATA_PROVIDER=fmp` + `FMP_API_KEY` | Financial Modeling Prep stable API；财报静态缓存、实时报价/新闻/8-K |
| **Yahoo**（默认/回退） | `DATA_PROVIDER=yahoo` | 免费 `yfinance`；FMP 受限标的（HTTP 402）自动回退 Yahoo 财报 |

FMP 付费 Starter 档建议：`FMP_DAILY_QUOTA=0`（禁用日封顶）、`FMP_REQUEST_INTERVAL=0.21`（约 286 次/分钟）。
本地缓存目录 `data/cache/fmp/`；GitHub Actions 已配置跨次运行缓存。

### 配置（一次性）

```bash
.venv/bin/pip install -r requirements.txt

# .env 示例（详见 .env.example）
GEMINI_API_KEY=...          # https://aistudio.google.com/apikey
DATA_PROVIDER=fmp
FMP_API_KEY=...
FMP_DAILY_QUOTA=0
FMP_REQUEST_INTERVAL=0.21
# 可选：GEMINI_MODEL=gemini-2.5-pro（默认 flash，省钱）
```

### 用法

```bash
# 完整分析（调用 Gemini）
PYTHONPATH=src .venv/bin/python scripts/analyze.py RKLB

# 带上买入动机
PYTHONPATH=src .venv/bin/python scripts/analyze.py 4063.T --note "半导体材料持仓"

# 只看硬指标（无 key 时自动进入）
PYTHONPATH=src .venv/bin/python scripts/analyze.py 6859.T --data-only

# 单股闪击（可选报告颗粒度）
PYTHONPATH=src .venv/bin/python lynch_cli.py --ticker RKLB --mode quarterly
```

日股加 `.T`（如 `4063.T`），美股直接写符号（如 `RKLB`、`AMD`）。

### 影子持仓 watchlist.yaml

`watchlist.yaml` 是**高优先级必看列表**：`--scope full` 时永远纳入分析、永远优先送 AI；
**不受 `MARKET` 过滤**（全市场漏斗仍按 `MARKET` 扫 SEC 美股抽样）。

```yaml
stocks:
  - ticker: "RKLB"
    name: "Rocket Lab"
    market: US
    tier: 1
    status: held    # held=已持有 / watch=观望 / avoid=黑名单物理隔离
    note: "NISA 5株"
```

- `status: avoid` — 流水线、盘中狙击、CLI **全部跳过**，用于已决定卖出的标的。
- `status: held` — 季报/年报 Prompt 会触发「持仓审视」铁律。
- 日股备用清单见 `watchlist-jp.yaml`（默认不自动纳入，可 `--tickers` 手动指定）。

### 输出读法

- **🟢/🟡/🔴** 三色灯：林奇量化排雷（绿=通过，黄=警惕，红=危险）。
- **PEG**：市盈率 ÷ 增长率。≤1 合理，≤0.5 极佳，>2 危险。
- **舆情安全网**：邮件数据块含「最新市场舆情监控」+ SEC 8-K；Gemini 对 FDA 召回、欺诈等**一票否决**。
- **政要巨鳄雷达**（FMP）：议员交易 + 伯克希尔/桥水/文艺复兴 13F 变动，简报前置一行异动提示。

> 铁律：拿不到真实财务数据前不下「值得投资」结论；不预测宏观，只回到公司基本面。

### 全自动后台流水线（GitHub Actions + 邮件）

**五种**定时简报 + **独立**盘中狙击 workflow：

| 模式 | 约 (JST) | 调 Gemini | 邮件标题前缀 |
|------|----------|-----------|--------------|
| **daily** 日报 | 周二~周六 06:00 | 否* | `【彼得林奇自选股监控】日报` |
| **weekly** 周报 | 周六 12:00 | 是 | `【彼得林奇深度分析】周报` |
| **monthly** 月报 | 月末最后交易日附近 | 是 | `【彼得林奇月度会诊】月报` |
| **quarterly** 季报 | 1/4/7/10 月末 | 是 | `【彼得林奇财报季会诊】季报` |
| **annual** 年报 | 12/31 | 是 | `【彼得林奇年终审视】年报` |

\*日报默认不调 Gemini；命中狙击条件（日跌>5% + PEG<0.5 + SBI 可买）时例外发加急信。
美股休市日（NYSE 节假日）日报自动跳过，不发「尸检式」空报。

**盘中狙击**（`sniper_realtime.yml`，与简报独立）：JST 周二~周六 23:00~次日 04:30 每 30 分钟扫描
`watchlist.yaml` 中美股，相对昨收跌>5% + 即时 PEG<0.5 时 Gemini 确认后发 `【🚨林奇深夜特快】` 邮件。

**双层漏斗（`--scope full`）：**

```
SEC 全美股抽样(~500) → 第一层硬指标(PEG≤1.5 或 净现金比≥30%, 负债≤50%)
  → 第二层 AI（`MAX_AI_ANALYSIS_COUNT` 默认 30；0=不限；held/watchlist 不占名额） → 邮件
```

FMP 冷启动时第一层用 `profile + ratios-ttm` 轻量筛选（约 2~3 次 API/股），避免一次拉全量财报。

**GitHub 配置（一次性）：**

1. **Secrets**：`GEMINI_API_KEY`、`FMP_API_KEY`、`SMTP_*`、`RECEIVER_EMAIL`；选填 `GEMINI_MODEL`
2. **Variables（可选）**：`DATA_PROVIDER=fmp`、`MARKET=US`、`UNIVERSE_SOURCES=us`、
   `MAX_UNIVERSE_SCAN`、`MAX_AI_ANALYSIS_COUNT`、`AI_SORT_KEY`
3. Actions → **Run workflow** 可选手动触发（mode / scope / market）

> 密钥只从环境变量读取，`.env` 不提交（已在 `.gitignore`）。

**本地跑同一条流水线：**

```bash
# 日报·仅必看列表
PYTHONPATH=src .venv/bin/python scripts/run_scheduled_analysis.py \
  --mode daily --scope watchlist --no-email

# 周报·全市场（先小范围试跑）
PYTHONPATH=src .venv/bin/python scripts/run_scheduled_analysis.py \
  --mode weekly --scope full --market US --max-universe 50 --no-email

# 盘中狙击本地测试
PYTHONPATH=src .venv/bin/python scripts/run_realtime_sniper.py --no-email
```

**Cron（UTC，见 `.github/workflows/run_analysis.yml`）**：`0 21 * * 1-5` 日报、
`0 3 * * 6` 周报、`0 21 28-31 * *` 月报、季报/年报见 workflow 注释。

技术细节见 [lynch-requirements.html](lynch-requirements.html)（浏览器打开）。

#### 怎么拿邮箱 SMTP 授权码？

- **Gmail**：`SMTP_SERVER=smtp.gmail.com`、`SMTP_PORT=587`。先在 Google 账号开启
  **两步验证**，再到「安全性 → 应用专用密码」生成 16 位密码，填入 `SMTP_PASSWORD`（不是登录密码）。
- **QQ 邮箱**：`smtp.qq.com`、端口 `465`。设置 → 账户 → 开启 SMTP → 生成**授权码**。
- **Outlook/163** 同理：开启 SMTP 服务后使用授权码。`SMTP_USERNAME` 填完整发件邮箱地址。
